SmartAgri — Setup Guide

Complete installation and run instructions for a new machine

v1.0  ·  June 2026
1
Install prerequisites

Download and install all three tools before doing anything else.

ToolMinimum versionWhere to download
Python3.11 or newerpython.org/downloads
Node.js (includes npm)18 LTS or newernodejs.org
PostgreSQL15 or 16postgresql.org/download
During PostgreSQL setup you will be asked to set a password for the postgres user. Write it down — you will need it in Step 4.

After installing, verify everything is on your PATH by opening Command Prompt and running:

REM Each line should print a version number, not an error python --version node --version npm --version
2
Extract the ZIP and install packages

Extract the ZIP file anywhere (e.g. C:\Projects\SmartAgri). Open Command Prompt inside that folder and run:

REM Install all Python packages (backend + ML service) cd backend pip install -r requirements.txt cd .. REM Install frontend packages cd frontend npm install cd ..
The pip install step downloads ML libraries (scikit-learn, XGBoost) and may take 2–5 minutes on first run. This is normal.
3
Create the database and import the data

First create an empty database named smartagri, then import the provided SmartAgri.sql file into it. Choose one method:

Method A — Command line recommended

Fastest and most reliable. Run two commands in order.

Method B — pgAdmin GUI

Use this if you prefer a graphical interface.

Method A — Command line

REM Step 1: create the empty database (enter your postgres password when prompted) psql -U postgres -c "CREATE DATABASE smartagri;" REM Step 2: import the SQL dump (adjust the path to where SmartAgri.sql is saved) psql -U postgres -d smartagri -f "C:\path\to\SmartAgri.sql"
If psql is not recognised, add PostgreSQL's bin folder to your PATH.
Example path: C:\Program Files\PostgreSQL\16\bin

Method B — pgAdmin

  • Open pgAdmin and connect to your local PostgreSQL server.
  • Right-click Databases → Create → Database… and name it smartagri.
  • Right-click the new smartagri database → Restore…
  • Set Format to Plain, browse to SmartAgri.sql, and click Restore.
Once the import finishes, all tables, data, and user accounts are ready. The application will detect this on startup and skip re-creating anything.
4
Create the .env configuration file

Inside the backend\ folder, create a new file named exactly .env (no other extension). Copy the content below and replace YOUR_PASSWORD with your PostgreSQL password from Step 1.

# backend\.env DATABASE_URL=postgresql://postgres:YOUR_PASSWORD@localhost:5432/smartagri SECRET_KEY=replace-this-with-any-long-random-string ALGORITHM=HS256 ACCESS_TOKEN_EXPIRE_MINUTES=30 SMARTAGRI_CORS_ORIGINS=http://localhost:5173

To generate a proper SECRET_KEY, run this in your terminal and copy the output:

python -c "import secrets; print(secrets.token_hex(32))"
The .env file is intentionally excluded from the ZIP because it contains your database password. Never share it or commit it to git.
5
Start the application

Double-click start-services.bat in the project root folder.

Do not run it as Administrator. Python and Node.js are installed in your user profile and are not accessible from an elevated (admin) prompt.

Three separate Command Prompt windows will open — one for each service. Wait until all three show they are ready (usually 10–15 seconds), then open your browser and go to:

http://localhost:5173

To stop all services when you are done, double-click stop-services.bat.

6
Log in

All accounts are already in the imported database. Use the admin account to access the full system:

admin@smartagri.lk
Admin@12345

Land Owner and Trader test accounts are also included in the database. Ask the person who sent you the ZIP for those credentials, or register new accounts from the Register page.
